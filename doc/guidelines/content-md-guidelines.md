# 全库 content.md 编写规范

面向 `assets/library/{category}/{slug}/content.md` 与 `assets/seed/library/items/**/*.json` 的维护约定。

## 分工

| 位置 | 内容 |
|------|------|
| **JSON `specs`** | 最多 **8** 条关键规格，面向详情页右侧参数表 |
| **content.md** | 叙述性正文、**文内配图**、参考链接；**禁止**文末「规格参数」与独立「配图参考」章节 |

## 篇幅与可读性

- 目标 **3–5 分钟阅读量**（约 **1200–2200 字**），分 4–6 个小节。
- 用 **加粗**、列表、短段拆分重点；避免百科粘贴与重复段落。
- 事实须可核对；发现错别字、张冠李戴须改正。

## content.md 推荐结构

```markdown
# 条目标题

开篇 1–2 段：是什么、为何值得了解。

![与上文直接相关的说明](https://境内可访问的直链)

## 背景与历史
……可在段落后补一张相关图……

## 特点 / 梗概 / 设计亮点
……

## 影响、周边与冷知识
……

## 参考与延伸阅读
- [来源名称](https://...)
```

**不要**使用 `## 配图参考` 把图片堆在文末。

## 配图原则

1. **来源**：维基共享、百科/媒体官方图床、新闻稿等均可；须 **中国大陆可稳定打开**（避免 404、防盗链失效、境外 CDN 超时）。
2. **相关性**：图片内容必须与该段文字一致；**对不上就换源，实在没有则不要配图**（宁缺毋滥）。
3. **位置**：写在对应段落**附近**（段前或段后），不要集中成附录章节。
4. **禁止**：同一 URL 批量套用到多个无关条目（如所有超跑共用一张 McLaren F1、所有变形金刚共用 Logo）。
5. **格式**：`![描述](URL)`；描述写清画面内容。

## 规格（specs）原则

- 每条 **≤8** 个键值，键名简短（中文优先）。
- 只保留用户最关心的字段。
- **不要**把 specs 写进 content.md。

## 禁止

- 大段未整理的百科重复粘贴。
- 文末 `## 规格参数`。
- 失效热链（如 `aka.doubaocdn.com` 等已 404 的占位图）。
- 与 JSON `summary` 完全重复的「资料汇编」块。

## 维护方式

直接编辑 `assets/library/{category}/{slug}/content.md` 与对应 `assets/seed/library/items/` 下 JSON，执行 `npm run build` 生成数据包。
