# Figma 设计

Figma 是浏览器优先的协作式界面设计工具，2016 年由 Dylan Field 与 Evan Wallace 创立。它以实时多人编辑、组件与变体（Variants）、Auto Layout 和 Dev Mode 著称，改变了设计团队从线框到交付的工作流——设计师不再「丢 PSD 过墙」，而是与产品、开发在同一画布上迭代。

## 背景与历史

Figma 脱胎于 WebGL 渲染实验，早期即坚持云端协作而非单机文件。2019 年推出插件生态，2022 年 Adobe 宣布收购（后于 2023 年因监管终止），期间 Figma 持续扩展 FigJam 白板、Variables 设计 Token 与 Code Connect 等能力。如今它已是 SaaS、移动端与 design system 团队的事实标准工具之一。

时间线与地域背景有助于理解它为何在特定年代走红，以及今日在收藏、实用或文化象征中的位置。


## UX 原则与产品影响

- **组件驱动**：Button、Input 等原子组件 + 变体属性，让设计系统可维护、可规模化。
- **Dev Mode**：开发者在 Figma 内查看间距、Token、CSS/Swift 片段，缩短设计—实现 gap。
- **协作默认**：评论、版本历史、分支（Branch）降低远程团队沟通成本。
- **可达性检查**：对比度、焦点顺序等插件与内置能力，推动 a11y 左移。

Figma 的普及也催生了「设计即代码」文化：Design Token 导出、Storybook 联动、shadcn 等复制粘贴组件模式，都假设设计源文件结构化且可共享。

## 冷知识

- Figma 名字来自拉丁语 *figma*（形状、形态），与界面「形」的概念呼应。
- 2012 年 Thiel Fellowship 资助了创始人，Figma 曾被视为「不可能在浏览器里跑专业设计」的赌注。
- Config 年度大会已成为设计行业风向标，常发布 Variables、Dev Mode 等重大功能。
